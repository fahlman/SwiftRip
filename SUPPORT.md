# Support

SwiftRip support happens through GitHub issues for now. The project is small, so useful reports are much easier to handle than broad "it does not work" reports.

## Before Opening an Issue

- Install the latest public SwiftRip release.
- Confirm your Mac meets the minimum macOS version listed on the latest release.
- Confirm the disc is a DVD with a `VIDEO_TS` folder.
- Confirm the output folder was selected through SwiftRip, not only typed or copied from another app.
- Try a second disc if possible.
- Check whether the issue also happens after quitting and relaunching SwiftRip.

## Useful Issue Details

Please include:

- SwiftRip version and build number from the app or release page.
- macOS version.
- Apple Silicon or Intel.
- Whether SwiftRip was installed from a DMG, updated by Sparkle, or built locally.
- DVD drive model if known.
- Whether the disc is encrypted, unencrypted, damaged, homemade, or commercial.
- What you clicked before the failure.
- The exact error shown by SwiftRip.
- The rip log if one was created.

Logs may include local file paths, mounted volume names, disc names, and output filenames. Redact anything personal before posting.

Do not attach copyrighted video, decrypted DVD contents, Apple account details, GitHub tokens, private keys, or app-specific passwords.

## What Support Can Cover

- SwiftRip install and launch problems.
- Sparkle update problems.
- Sandbox permission and folder bookmark problems.
- DVD detection problems.
- Rip start, progress, cancel, and failure behavior.
- Bundled tool packaging problems.
- Reproducible crashes or hangs.
- Incorrect or confusing app messages.

## What Support Cannot Cover

- Legal advice.
- Help copying media you do not have the legal right to copy.
- Blu-ray ripping.
- General HandBrake preset tuning beyond SwiftRip's built-in workflow.
- Repairing damaged discs or DVD drives.
- Support for locally modified binaries unless the same issue reproduces from source or an official release.

## Security Issues

Do not report vulnerabilities in a public support issue. Follow [SECURITY.md](SECURITY.md) instead.
